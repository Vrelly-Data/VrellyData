## Changes

### 1. Email Stats Aggregation Fix

**Problem**: When uploading a contact-level CSV (e.g., 164 rows across 2 campaigns), each row overwrote the previous one, leaving stats at ~1 instead of the true totals.

**Fix**: The upload hook (`useEmailStatsUpload.ts`) now aggregates contact-level rows by campaign ID/name before writing to the database. The preview dialog (`EmailStatsUploadDialog.tsx`) also groups rows so it shows "Import 2 Result(s)" instead of 164.

**Before**:
- 164-row CSV → 164 individual DB updates → final stats = last row's values (~1)
- Preview showed "Import 164 Result(s)"

**After**:
- 164-row CSV → aggregated to 2 campaigns → 2 DB updates → correct totals
- Preview shows "Import 2 Result(s)"

### 2. Webhook Messaging Removal

**Problem**: The Messages Sent and Replies popover cards displayed "Real-time via webhooks" and "Enable webhooks for LinkedIn tracking" footer text, referencing functionality that is no longer used.

**Fix**: Removed the conditional `border-t` footer blocks from `messagesTooltipContent` and `repliesTooltipContent` in `PlaygroundStatsGrid.tsx`.
